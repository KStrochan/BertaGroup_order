# Реєстрація клієнтів + історія замовлень

## Що додано

- `lib/store.mjs` — просте збереження даних у JSON-файлах (`data/users.json`, `data/orders.json`), без зовнішньої БД.
- `lib/auth.mjs` — хешування паролів (scrypt) і підписані сесії (cookie).
- `server.mjs` — оновлений: нові ендпоінти + збереження кожного замовлення в акаунт клієнта.
- `public/index.html`, `public/app.js`, `public/styles.css` — оновлені: кнопка "Увійти", модалка входу/реєстрації, "Мій акаунт" з історією замовлень.
- `.env.example`, `.gitignore` — оновлені.

## Як застосувати у своєму репозиторії

1. Скопіюйте файли з цього архіву у відповідні місця репозиторію (заміняють існуючі):
   - `server.mjs` → корінь репо
   - `lib/` → корінь репо (нова папка)
   - `public/index.html`, `public/app.js`, `public/styles.css` → в `public/`
   - `.env.example`, `.gitignore` → корінь репо (просто заміните вміст, або перенесіть різницю вручну, якщо у вас там є ще щось своє)

2. У свій `.env` додайте рядок (згенеруйте один раз і збережіть назавжди — інакше клієнти будуть розлогинюватись при кожному перезапуску сервера):
   ```
   SESSION_SECRET=<довгий випадковий рядок>
   ```
   Згенерувати можна командою:
   ```bash
   node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

3. Запустіть як завжди:
   ```bash
   npm start
   ```

4. Дані клієнтів і замовлень зберігатимуться у папці `data/` (створюється сама). Вона вже додана в `.gitignore` — **ніколи не комітьте цю папку**, там паролі (у хешованому вигляді) і персональні дані клієнтів.

## Нові API-ендпоінти

| Метод | Шлях | Опис |
|---|---|---|
| POST | `/api/auth/register` | `{phone, password, company, contact}` |
| POST | `/api/auth/login` | `{phone, password}` |
| POST | `/api/auth/logout` | — |
| GET  | `/api/auth/me` | Поточний користувач (або `null`) |
| GET  | `/api/orders` | Історія замовлень поточного користувача |

`/api/order` тепер додатково зберігає замовлення в `data/orders.json`, прив'язане до `userId`, якщо клієнт залогинений (для гостей `userId: null`, замовлення все одно йде в Telegram як і раніше).

## Важливо перед тим, як показувати клієнтам

- **Постійний диск.** Якщо деплоїте на Render/Railway/подібне — переконайтесь, що увімкнений persistent disk для папки `data/`, інакше дані зникнуть при кожному передеплої.
- **Це проста файлова "база"**, розрахована на десятки-сотні клієнтів. Якщо клієнтська база виросте на порядки — варто буде перейти на справжню БД (SQLite/Postgres); структура коду (`lib/store.mjs`) зроблена так, щоб це можна було замінити без переписування решти сервера.
- **Сесії не відкликаються достроково** (наприклад, немає кнопки "вийти з усіх пристроїв") — токен просто спливає через 7 днів. Для B2B-каталогу такого рівня цього достатньо, але майте на увазі.
